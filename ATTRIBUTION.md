## Authorship Breakdown

### System architecture and design
Authored by me. The README sections up to and including "Architecture" were written 
by me and used verbatim as the prompt to Claude Code. Specifically, I designed:
- the rerouter pattern
- the backend abstraction with UNKNOWN as a sentinel
- letting the agent decide on the best tool to use in MCP fashion instead of using RAG (reason below)
- the experimental setup / evaluation methodology

### Initial code generation
Claude Code generated the initial implementation of the all the files  in /src based on 
my architectural prompt, as well as the files in /tests. 

### My modifications and additions
After initial generation, I made the following changes myself to fix bugs, 
strengthen the architecture, and improve observability:

- **`local_llm.py`** — The original model was to retrive tools using RAG, and 
  allow the model to retrieve tool usage based on the descriptions generated by claude.
  Testing out on various thresholds, I have learned that "get_current_weather" had a similarity
  score of 0.39 when retrieving for "Whats the weather in Tokyo?", while when reducing the threshold
  too low, the threshold caught random tools in its radar. Thus, I have allowed the model to decide
  based on the top three tools generated, and it has so far given me 100% success rates, though it might 
  be different when scaled.

- **Backend rerouter** — added an `UNKNOWN` backend as a sentinel to verify the 
  rerouter logic was actually firing and not silently defaulting. This caught 
  the bug where the model straightly outputed that it didn't know the answer, and that 
  it can not do these kind of tasks.

- **Logging** — added structured logs at server.py to debug implicit behavior in the part, and tried to find out where it had
gone wrong.

- **catalog.py** — I removed unnecessary function in "find_best_match". 
  This was specifically for the RAG prototype of finding best skill, where
  I tried to use cosine similarity of best description. I realized that it was
  very hard to decide the cosine similarity threshold of determining whether there
  was a suitable tool or not, so I made a hybrid model of retriving top 3 best matches
  and trying to find a match.
- **toolsmith.py** - I explicitly had to change and iterate the prompt so that the tools were not
  not generating multiple files or a whole codebase. Further, I had to make sure that it was only generating
  one python functions. Further, I had to add direct error handling when the toolsmith was given impossible directions. 
  The final change was to add the validating roop so that it did the repair and did waht I had instructed.
